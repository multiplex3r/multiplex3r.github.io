XSS for beginners!

2014-11-08

This post is going to go into one of the most common web attacks and how you can try it for yourself. The attack is the notorious Cross-site Scripting attack. I say notorious, because it’s 2014. Why are we still seeing this out in the wild? Sanitise your inputs people!

Cross-site Scripting

In the Web 2.0 world we live in today, there are not many people who are in infosec and haven’t heard about XSS. So, what exactly IS cross-site scripting you may ask? Cross-site scripting or XSS is a vulnerability which allows an attacker to inject Javascript into web pages which is then executed by the users browser.

A typical use case for XSS, would be to steal session cookies; allowing the attacker to log-in to a web application as the victim. Or to load some other payload or malware on to the victims machine.

XSS is bad.

Here is an example of some PHP code that is vulnerable to XSS via the username parameter.

<?php
$username = $_REQUEST ['username'];
?>

<html>
    <title>
        XSS Example
    </title>
    <body>
        Hello, <?php echo $username; ?>!
    </body>
</html>

To have a play with the above code, have a system with PHP installed (you can get it from here). Then save the code snippit in a new directory to a file called index.php. You will then be able to execute the following command php -S 127.0.0.1:8080 -t . from inside the new directory and browse to it.

Modify the url to something similar changing ‘multiplex3r’ to your username or XSS payload of choice.

This will allow you to change the username parameter to <script>alert("XSS")</script> to demo a basic XSS payload..

Remember, hack ethically and don’t pwn stuff you don’t own!

NOTE: Please use your $(browser) no-script addon. It’s really great and stops the blind execution of Javascript!
