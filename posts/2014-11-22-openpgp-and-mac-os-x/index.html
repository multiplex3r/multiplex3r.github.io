OpenPGP and Mac OS X

2014-11-22

Over the last few months I’ve been experimenting with Mac OS X’s Mail.app and GPGSuite for Mac. During the period of the Beta and Developer previews of OS X Yosemite, GPGMail was unusable and crashed Mail. This sent me on a search for either a better Mail client on OS X or a better way to do OpenPGP on OS X. This is a post about my journey to the command line and back again.

People keep telling me Mac Mail is a pretty ordinary mail client. I’ve stuck with it for a while because it’s integrated into OS X so well. Also, it’s not fixing something by throwing yet more software at a problem. Plus (and this is a big plus) it’s pretty. On Retina screens it looks amazing. I’ve used a lot of the other mail clients out there, and there isn’t one I’ve found that looks nearly as good.

So Mac Mail is pretty and it has the guys from GPGTools doing it’s OpenPGP addon. Sounds great right? Perfect mail client? Wrong. I was sending signed email, no problem if you had Mail+GPGTools you could see my valid signature automatically. I could send encrypted email WITH attachments. If you had Mail+GPGTools it would work a treat!

Unfortunately, or fortunely ;) not everyone uses Mail+GPGTools. So I set about finding a better way. I dabbled with command line GPGTools. But that felt too strange coming from the work flow I had been using where there is almost no extra effort to sign/encrypt. And I liked that work flow. So CLI was out.

THUNDERBIRD. I used to use Linux as a primary OS before I got my MBP. SURELY Thunderbird was still a valid option. Pop Enigmail into it and we’re off right? Wrong. Thunderbird looks so bad after using Mail for what seems like forever. And the formatting of text? I think it’s important that it looks good being that we communicate with people so much over email. Unfortunately Thunderbird just didn’t check this box for me.

Was I doomed to choose between Security or Usability? Well I was reluctantly going to go with Thunderbird until I came across THIS post about binding keys to specific OpenPGP service functions. Hmm this looks promising.

I followed the guide assigned some keybinds and then BAM. I can now sign my mail inline without using the GPGMail addon. And to top it off, it works on encrypted files. I can email anyone with any client and it doesn’t tear the protocol apart trying to get it to them.

I get security and usability :)
